Gestion des requêtes et des pins d'un microcontrôleur

🌐 Communication entre le site web et le microcontrôleur
• Un code est mis en place pour déterminer l'état des pins et contrôler les appareils.
• Un point d'accès WLAN permet la communication entre le site de gestion et le PC sur le même réseau.

📡 Gestion des requêtes REST API
• Différents types de requêtes sont envoyés au REST API pour contrôler les composants.
• Les requêtes sont du type fetch("https:192.168.1.1/eteindre/lampe1").

🕹 Contrôle des pins du microcontrôleur
• Le microcontrôleur dispose de routes et d'API pour gérer les différents pins.
• Les requêtes envoyées depuis le REST API permettent de vérifier l'état des composants.
• Les requêtes incluent des actions telles que ouvrir une porte ou éteindre une lampe.
• Un site web est mis en ligne pour gérer les pins et permet la communication avec le PC.

Présentation générale :

Ce système permet de gérer et de contrôler des dispositifs IoT (Internet des objets) à l'aide d'un microcontrôleur ESP8266 ou compatible Arduino connecté en Wi-Fi. Il repose sur une architecture comprenant un site web, un serveur REST API, et un microcontrôleur qui interagit avec le matériel connecté (comme des lumières, portes, etc.).

Les composants principaux :

  1. Site web :
    • Sert d'interface utilisateur pour gérer les dispositifs connectés.
    • Envoie des requêtes HTTP via une API REST pour exécuter des actions (allumer/éteindre, vérifier l’état des dispositifs, etc.).
  1. PC ou Raspberry Pi :
    • Héberge le serveur web et l'API REST.
    • Assure la communication entre le site web et le microcontrôleur IoT.
    • Peut également servir de point d'accès Wi-Fi pour établir un réseau local.
  1. Microcontrôleur (ESP8266 ou Arduino compatible Wi-Fi) :
    • Commande le matériel connecté via ses pins (broches).
    • Inclut un serveur web intégré pour recevoir les requêtes du serveur REST.
    • Dispose de routes API spécifiques pour gérer les broches du microcontrôleur.

Flux de communication :

  1. Du site web vers l’API REST :
    • Le site web envoie des requêtes HTTP au serveur REST hébergé sur le PC ou Raspberry Pi.
    • Exemple de requêtes :
      • fetch("http://192.168.1.1/eteindre/lampe1") : Éteint une lampe spécifique.
      • fetch("http://192.168.1.1/ouvrir/porte1") : Ouvre une porte spécifique.
    • L’API REST vérifie l’identité de l’utilisateur (via un passkey) avant de transmettre les commandes.
  1. De l’API REST vers le microcontrôleur :
    • L’API REST transmet les commandes au microcontrôleur (ESP8266 ou Arduino).
    • Les commandes peuvent inclure :
      • pin-on/D1,D2 : Active les broches D1 et D2.
      • pin-off/D2 : Désactive la broche D2.
      • pin-status/all : Récupère l’état de toutes les broches.
    • Cela permet de centraliser le contrôle des dispositifs connectés sans devoir reprogrammer le microcontrôleur pour chaque mise à jour.
  1. Du microcontrôleur vers le matériel connecté :
    • Le microcontrôleur exécute les commandes reçues pour interagir avec le matériel (par exemple, allumer une LED, activer un relais, ou ouvrir une porte).

Avantages du système :

  1. Modularité :
    • Les fonctionnalités peuvent être modifiées ou étendues sans avoir à recompiler le code du microcontrôleur.
  1. Évolutivité :
    • Le système peut facilement gérer de nouveaux dispositifs en ajoutant des routes dans l’API REST.
  1. Supervision en temps réel :
    • Les commandes comme pin-status permettent de vérifier instantanément l’état des dispositifs connectés.

Cas d’utilisation possibles :

  1. Domotique :
    • Contrôle à distance des lumières, portes, et autres appareils dans une maison connectée.
  1. Prototypage IoT :
    • Permet de tester rapidement les interactions entre matériel et logiciel via une interface REST.
  1. Automatisation en réseau :
    • Contrôle simultané de plusieurs dispositifs sur le même réseau Wi-Fi.

Résumé des types de requêtes API :

  • pin-on/D1,D2 : Allume les broches D1 et D2.
  • pin-off/D2 : Éteint la broche D2.
  • pin-status/all : Renvoie l’état de toutes les broches.
  • pin-status/D1 : Renvoie l’état de la broche D1.

Ce système est à la fois pratique, sécurisé et adaptable, idéal pour des projets de domotique ou de prototypage IoT.
